# Security policy

## Report privately

Use GitHub's **Report a vulnerability** flow for this repository when a documentation
site, downloadable fixture, build script, dependency, or deployment issue could expose
credentials, execute untrusted code, misdirect a download, or weaken PatchLab's safety
boundary. Do not include secrets or exploit details in a public issue.

For ordinary factual errors, broken links, accessibility defects, or rendering bugs,
use the public documentation issue template.

## Response

Maintainers will acknowledge a private report when repository notifications permit,
triage its scope, and coordinate disclosure after a correction or mitigation exists.
No response-time or bounty promise is made. Supported security scope is the current
default branch and the currently published documentation artifact.
