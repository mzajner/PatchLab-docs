Start with the question the patch must answer, not the most impressive block name.

| Need | Begin with |
| --- | --- |
| Create an audible source | [Generators](/guides/block-families/generators/) |
| Shape tone or split bands | [Filters & EQ](/guides/block-families/filters-and-eq/) |
| Control level or add nonlinearity | [Dynamics & Distortion](/guides/block-families/dynamics-and-distortion/) |
| Add delay, pitch, movement, or space | [Time & Motion](/guides/block-families/time-and-motion/) |
| Move parameters over time | [Modulators](/guides/block-families/modulators/) |
| Respond to notes, clocks, or decisions | [Notes & Sequencing](/guides/block-families/notes-and-sequencing/) |
| Split, combine, nest, or change evaluation | [Routing & Structure](/guides/block-families/routing-and-structure/) |
| Observe rather than intentionally process | [Meters & Analysis](/guides/block-families/meters-and-analysis/) |
| Work on magnitude/phase frames | [Spectral](/guides/block-families/spectral/) |
| Add non-audio face media | [Visual](/guides/block-families/visual/) |

After choosing a family, compare exact ports, units, defaults, latency, and CPU class in
the generated [Block reference](/reference/blocks/). Audition with a controlled source,
then measure the claim that matters.
